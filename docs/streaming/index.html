Overview

Streaming provides stream producers (sources), stream consumers (sinks) and stream transformers (flows). The decoupled streaming abstractions are exposed in a convenient way in the Stream module.

The following features are provided:

  • Low memory usage: large or infinite streams can be computed in constant and small space. Buffering is possible when needed.
  • Blazing fast: all models were designed with performance at core. See benchmarks for detailed comparison with other libraries.
  • Resource safety: resources in streaming pipelines are allocated lazily and released as early as possible.
  • Flexibility: both push-based and pull-based models are implemented.

Quickstart

Install using opam

The library can be installed with OPAM: opam install streaming.

Install using esy

Add "@opam/streaming": "0.1" to dependencies in your package.json file.

Use with dune

To start using Streaming in your project add it to libraries in the dune file of your target.

(executable
  (public_name main)
  (libraries streaming))

Use in utop

Fire up utop and write:

#require "streaming";;
open Streaming;;
Stream.(stdin |> take_while ((<>) "stop") |> stdout);;

Examples

It is recommended to open the top-level Streaming module in your projects. This will expose the main streaming primitives such as Streaming.Stream and Streaming.Sink.

open Streaming

(* Print lines from stdin, ignoring empty input. *)
let stdin_echo =
  Stream.stdin
  |> Stream.filter ((<>) "")
  |> Stream.map (fun line -> "You wrote: " ^ line)
  |> Stream.each print_endline

Concepts

  • Streams - push-based producers of values.
  • Sources - pull-based incremental producers of values.
  • Sinks - push-based consumers of values.
  • Flows - push-based transformation operations on values.

Troubleshooting

Configuration

  • Error: Unbound module Streaming - the library was not loaded into the environment. See the Quickstart section for more information.
  • Incorrect Stream module - if you forget to open Streaming your might start using the Stream module from the standard library. This will result in Unbound value and type mismatch errors.
  • My stream pipeline never terminates - this indicates that one of your producers or transformers produces an infinite stream.
